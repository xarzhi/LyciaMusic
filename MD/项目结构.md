# LyciaMusic 项目结构文档

本文档旨在帮助开发者和AI快速理解 LyciaMusic 项目的架构、文件组织、关键数据结构及核心工作流。

## 1. 项目概览

**LyciaMusic** 是一个基于 **Tauri** 构建的现代本地音乐播放器。它结合了 Web 前端的灵活性和 Rust 后端的高性能。

- **核心功能**：本地音乐库管理、多格式播放、动态背景主题、歌词显示、播放历史统计。
- **设计理念**：极致的视觉体验（玻璃拟态、动态流光）、高性能（虚拟列表、Rust 后端扫描）、数据隐私（本地 SQLite）。

## 2. 技术栈 (Tech Stack)

### 前端 (Frontend)
- **框架**: [Vue 3](https://vuejs.org/) (Composition API, Script Setup)
- **构建工具**: [Vite](https://vitejs.dev/)
- **语言**: [TypeScript](https://www.typescriptlang.org/)
- **样式**: [Tailwind CSS v4](https://tailwindcss.com/) (配合 PostCSS)
- **路由**: [Vue Router](https://router.vuejs.org/)
- **图标**: [Lucide Vue Next](https://lucide.dev/)
- **性能优化**: `@tanstack/vue-virtual` (虚拟列表)

### 后端 (Backend)
- **框架**: [Tauri v2](https://tauri.app/)
- **语言**: [Rust](https://www.rust-lang.org/)
- **数据库**: [Rusqlite](https://github.com/rusqlite/rusqlite) (SQLite)
- **异步运行时**: [Tokio](https://tokio.rs/)
- **核心依赖**:
  - `rodio`: 音频解码与播放
  - `walkdir`: 高效文件扫描
  - `id3`, `metaflac`: 音频元数据读取

## 4. 核心架构与模块详解

### 4.1 前端核心 (src/composables/player.ts)
这是前端的"大脑"。它不只管理播放状态，还充当了状态管理库（这就是为什么你没看到 Pinia 的原因）。
- **状态管理**: 维护 `currentSong`, `isPlaying`, `playQueue`, `volume` 等。
- **持久化**: 负责调用 Tauri 命令保存/读取数据。
- **通信**: 监听 Rust 发来的事件（如扫描进度）。
- **功能**: 提供 `play()`, `pause()`, `next()`, `scanLibrary()` 等全局方法。

### 4.2 后端核心 (src-tauri/src/lib.rs & music/)
Rust 端主要作为"服务层"，通过 Tauri Command 暴露能力。
- **Library Module (`music/`)**: 负责扫描文件夹、解析 ID3/FLAC 标签、构建文件树。
- **Database Module (`database.rs`)**: 封装了 SQLite 操作，保证数据一致性。
- **Player Module (`player.rs`)**: 封装 `rodio`，处理音频流的解码和输出设备管理。

## 5. 关键数据结构 (Key Data Structures)

### 5.1 数据库 Schema (SQLite)

主要包含三张核心表：

1.  **songs** (存储歌曲元数据)
    ```sql
    CREATE TABLE songs (
        id INTEGER PRIMARY KEY,
        path TEXT NOT NULL UNIQUE, -- 文件绝对路径作为唯一标识
        title TEXT,
        artist TEXT,
        album TEXT,
        duration INTEGER,
        cover_path TEXT,
        added_at INTEGER,          -- 入库时间
        -- Audio Quality (v1.1.1+)
        bitrate INTEGER,
        sample_rate INTEGER,
        bit_depth INTEGER,
        format TEXT,
        file_size INTEGER,
        file_modified_at INTEGER
    );
    ```

2.  **library_folders** (库目录管理)
    ```sql
    CREATE TABLE library_folders (
        path TEXT PRIMARY KEY,
        added_at INTEGER
    );
    ```

3.  **play_history** (播放历史与统计)
    ```sql
    CREATE TABLE play_history (
        id INTEGER PRIMARY KEY,
        song_path TEXT NOT NULL,
        played_at INTEGER NOT NULL,
        event TEXT DEFAULT 'play', -- 'play', 'skip' 等
        played_seconds INTEGER DEFAULT 0,
        song_id INTEGER            -- 关联 songs.id
    );
    ```

### 5.2 TypeScript 接口 vs Rust 结构体

前后端数据结构通过 Serde 序列化保持一致：

**TS Interface (`src/types/index.ts`):**
```typescript
export interface Song {
  id?: number;
  path: string;
  title: string;
  artist: string;
  album: string;
  duration: number;
  cover?: string;
  // ... 其他音质字段
}
```

**Rust Struct (`src-tauri/src/music/types.rs`):**
```rust
#[derive(Serialize, Clone, Debug)]
pub struct Song {
    pub id: Option<i64>,
    pub path: String,
    pub title: String,
    pub artist: String,
    // ...
}
```

## 6. 关键工作流 (Key Workflows)

### 6.1 音乐库扫描流程
1.  **用户操作**: 在设置页添加文件夹 -> 点击"扫描"。
2.  **Frontend**: 调用 `usePlayer.addLibraryFolder` -> 调用 Tauri cmd `add_library_folder`。
3.  **Backend (`scanner.rs`)**:
    *   使用 `walkdir` 递归遍历目录。
    *   过滤支持的音频格式 (`mp3`, `flac`, `wav`, `m4a` 等)。
    *   **并发解析**: 使用 `Rayon` 或 `Tokio` 并发读取文件元数据。
    *   **入库**: 将元数据写入 `songs` 表 (使用 `INSERT OR IGNORE` 或 `REPLACE`)。
4.  **UI 更新**: 扫描完成后，前端刷新列表，`SongList` 组件通过虚拟列表渲染成千上万条数据。

### 6.2 播放流程
1.  **Frontend**: 用户点击歌曲 -> `player.ts` 的 `play(song)` 被调用。
2.  **Command**: 调用 Tauri cmd `play_audio(path)`.
3.  **Backend (`player.rs`)**:
    *   `rodio` 加载音频文件。
    *   开始解码并输出到默认音频设备。
4.  **封面与歌词**:
    *   前端同时请求封面: `get_song_cover(path)`.
    *   前端同时请求歌词: `get_song_lyrics(path)`.

### 6.3 动态背景 (Color Extraction)
1.  **触发**: 歌曲切换时，`usePlayer` 获取新封面。
2.  **Analysis (`colorExtraction.ts`)**:
    *   在 Canvas 上绘制封面。
    *   提取主色调 (Dominant Color) 和调色板。
3.  **Appply**:
    *   **流光模式 (Flow)**: 使用 CSS 动画和提取的颜色更新 `GlobalBackground.vue` 的渐变层。
    *   **模糊模式 (Blur)**: 设置静态背景图并应用 CSS `backdrop-filter: blur()`.

## 7. 开发者指南

-   **新增页面**: 在 `src/views` 新建 `.vue` -> 在 `src/router/index.ts` 注册路由。
-   **新增后端命令**:
    1.  在 `src-tauri/src/music/` 或对应模块编写 Rust 函数，标记 `#[tauri::command]`.
    2.  在 `src-tauri/src/lib.rs` 的 `tauri::generate_handler!` 宏中注册。
    3.  前端使用 `invoke('command_name', args)` 调用。
-   **数据库变更**: 修改 `src-tauri/src/database.rs`，并在 `DbState::new` 中添加迁移逻辑 (Migration Logic)。
