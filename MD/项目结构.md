# LyciaMusic 项目结构文档

本文档基于当前仓库代码（`main` 工作区现状）整理，目标是帮助开发者快速理解项目的目录组织、模块职责、数据结构与核心工作流。

- 项目名称：LyciaMusic
- 当前前端版本：`package.json` -> `1.2.0`
- 技术形态：Tauri v2 桌面应用（Vue 3 前端 + Rust 后端）
- 主要适配平台：Windows（托盘、媒体控制、窗口行为均已接入）

---

## 1. 技术栈与关键依赖

### 1.1 前端

- 框架：Vue 3（Composition API）
- 构建：Vite 6
- 语言：TypeScript
- 样式：Tailwind CSS 4 + PostCSS
- 路由：Vue Router 4
- 组件/图标：`lucide-vue-next`
- 性能相关：`@tanstack/vue-virtual`（虚拟列表能力）
- Tauri 前端接口：
  - `@tauri-apps/api`
  - `@tauri-apps/plugin-dialog`
  - `@tauri-apps/plugin-opener`

### 1.2 后端（Rust）

- 框架：Tauri v2
- 数据库：SQLite（`rusqlite`，bundled）
- 音频：`rodio` + `cpal`
- 媒体控制：`souvlaki`
- 元数据读取：`lofty`
- 文件扫描：`walkdir`
- 并发与运行时：`tokio`
- 其他：`image`、`regex`、`sha2`、`hex`、`base64` 等

说明：当前代码并未使用 `id3` / `metaflac` 作为直接依赖，元数据统一通过 `lofty` 读取。

### 3.1 前端职责划分

- `player.ts`：
  - 管理播放、队列、视图切换、筛选排序、收藏/歌单操作
  - 与 Rust 命令交互（`invoke`）
  - 监听后端事件（`player:play/pause/next/prev`、`seek_completed`）
  - 负责本地持久化（大量状态通过 `localStorage` 保存）
- `playerState.ts`：
  - 维护全局响应式源数据（歌曲列表、当前歌曲、主题配置、目录树等）
- `lyrics.ts`：
  - 解析 LRC，计算当前高亮行（考虑 `AUDIO_DELAY`）

### 3.2 当前路由

- `/`：Home（本地音乐主视图，含统计入口）
- `/favorites`：收藏
- `/recent`：最近播放
- `/artists`：艺术家页（懒加载）
- `/albums`：专辑页（懒加载）
- `/settings`：设置页

### 4.1 `lib.rs` 启动流程

1. 初始化插件：
   - 单实例插件（第二次启动拉起主窗口）
   - dialog / opener
2. 初始化状态：
   - `DbState`（数据库连接）
   - `PlayerState`（播放器线程通信）
3. 初始化图片并发限制：
   - `ImageConcurrencyLimit(Semaphore::new(4))`
4. 启动时清理缓存：
   - `run_cache_cleanup(...)`
5. 构建系统托盘：
   - 显示主窗口 / 退出
6. 注册 Tauri commands（见第 7 节）

## 5. 数据层与数据库结构

数据库文件位于 Tauri app data 目录：`library.db`。

### 5.1 核心表

1. `songs`
- 主键：`id`
- 唯一键：`path`
- 主要字段：`title/artist/album/duration/cover_path`
- 音质字段：`bitrate/sample_rate/bit_depth/format/file_size`
- 时间字段：`added_at/file_modified_at`

2. `library_folders`
- 维护“音乐库目录”
- 字段：`path`（PK）、`added_at`

3. `sidebar_folders`
- 维护“侧边栏目录树”
- 字段：`path`（PK）、`added_at`

4. `play_history`
- 播放行为统计
- 字段：`song_path/song_id/played_at/played_seconds/event`

### 5.2 迁移策略（`database.rs`）

应用启动时按列存在性做增量迁移（`PRAGMA table_info` + `ALTER TABLE`），避免破坏已有用户数据。

已处理的迁移方向包括：
- `songs` 增加音质与文件信息字段
- `play_history` 增加 `played_seconds` 与 `song_id`
- 索引创建：`idx_songs_added_at`、`idx_play_history_played_at`、`idx_play_history_song_id`

## 6. 核心工作流

### 6.1 扫描与增量更新

入口：`scan_music_folder` / `scan_library`

流程概述：
1. 用 `WalkDir` 扫描目录（当前过滤：`mp3`/`flac`/`wav`）
2. 与数据库快照比较（`file_modified_at + file_size`）
3. 分为新增、更新、删除三类
4. 事务批量提交（INSERT/UPDATE/DELETE）
5. 返回最新歌曲列表

说明：当前实现为 `spawn_blocking + 同步扫描`，不是 Rayon 并发扫描模型。

### 6.2 播放流程

1. 前端调用 `play_audio`，并同步更新当前 UI 状态
2. 后端 `player.rs` 通过 channel 向播放线程发送 `AudioCommand`
3. 播放线程管理 `rodio::Sink`：
   - `Play/Pause/Resume/Seek/SetVolume/SetDevice`
4. 通过采样计数估算播放进度（`get_playback_progress`）
5. 媒体按键事件从 `souvlaki` 回传前端（`player:*` 事件）

### 6.3 歌词流程

1. 前端 `lyrics.ts` 在切歌时调用 `get_song_lyrics`
2. 后端优先从音频标签读歌词，失败则尝试同名 `.lrc`
3. 前端解析时间轴并驱动滚动/高亮

### 6.4 统计流程

- `record_play` 写入 `play_history`
- `get_library_stats`：曲库总体指标
- `get_quality_distribution`：Hi-Res / SQ / HQ / Other
- `get_format_distribution`：FLAC/MP3/APE/WAV/AAC/Other
- `get_behavior_stats`：时间范围内播放次数、时长、Top 歌曲/歌手/专辑、时段分布

## 8. 配置与状态管理

### 8.1 前端状态来源

存在两组设置来源（需注意）：

1. `src/composables/settings.ts`
- 偏通用应用设置（如最小化到托盘、关闭到托盘、显示音质标记）

2. `src/composables/playerState.ts` 的 `settings`
- 偏播放器/主题/侧边栏显示控制（动态背景、自定义皮肤、侧栏模块开关等）

这两套设置都走 `localStorage`，维护时需避免字段重复或冲突。

### 8.2 Tauri 关键配置（`src-tauri/tauri.conf.json`）

- `productName`: `Lycia Music`
- `identifier`: `com.lover.lyciamusic`
- 主窗口：无系统边框（`decorations: false`）
- 资源协议：开启 `assetProtocol`
- 构建脚本：
  - dev 前置：`npm run dev`
  - build 前置：`npm run build`

## 9. 维护建议（针对当前代码）

1. 统一设置模型
- 当前存在 `useSettings` 与 `playerState.settings` 双配置源，建议抽象合并或明确边界。

2. 明确扫描格式策略
- `scanner.rs` 与工具箱支持格式不完全一致（前者 `mp3/flac/wav`，后者还含 `m4a/ogg`），建议文档与代码统一。

3. 编码与注释清理
- 部分源码存在乱码注释，不影响运行但影响维护，可统一转 UTF-8 并清理注释。

4. 命令分层
- `player.ts` 体量较大，建议按“播放、曲库、目录树、歌单、统计”进一步拆分 composables。
