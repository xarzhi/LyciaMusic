# LyciaMusic 项目结构文档

> 更新日期：2026-02-06
> 适用版本：LyciaMusic v1.2.0 (Tauri v2)

## 1. 项目概览

LyciaMusic 是基于 Tauri v2 + Vue 3 的本地音乐播放器，强调高性能本地库管理与沉浸式播放体验。

- 本地音乐库扫描与管理（库目录与侧边栏目录分离）
- 多格式播放、输出设备切换、播放进度与音量控制
- 封面/歌词解析与动态背景色提取
- 播放历史与统计图表
- 工具箱：批量重命名与规则清洗

## 2. 技术栈

### 前端 (Frontend)

- 框架：Vue 3 (Composition API, `<script setup>`)
- 构建：Vite 6
- 语言：TypeScript 5
- 样式：Tailwind CSS 4 + PostCSS
- 路由：Vue Router 4
- 图标：Lucide Vue Next
- 性能：`@tanstack/vue-virtual` (虚拟列表)
- Tauri API：`@tauri-apps/api` + `@tauri-apps/plugin-dialog` + `@tauri-apps/plugin-opener`

### 后端 (Backend)

- 框架：Tauri v2
- 语言：Rust 2021
- 数据库：Rusqlite (SQLite)
- 音频：Rodio + cpal
- 元数据：lofty
- 扫描：walkdir
- 图片处理：image
- 异步：tokio
- 媒体控制：souvlaki
- 工具：regex
- 插件：`tauri-plugin-single-instance` / `tauri-plugin-opener` / `tauri-plugin-dialog`

## 4. 前端核心模块

- `src/composables/player.ts`：播放控制、库扫描、播放队列与 Tauri 命令调用的核心编排
- `src/composables/playerState.ts`：全局响应式状态（当前曲目、队列、主题色等）
- `src/composables/colorExtraction.ts`：封面主色提取与动态背景
- `src/composables/lyrics.ts`：歌词解析与滚动控制
- `src/components/song-list/`：列表、表格与主面板（虚拟列表渲染）
- `src/components/player/`：播放详情、歌词、队列侧栏
- `src/components/settings/`：设置页面与工具箱步骤（`SettingsToolbox.vue` / `ToolboxStep*.vue`）
- `src/components/statistics/`：统计页与图表卡片
- `src/views/`：首页、收藏、最近、艺术家、专辑、设置等页面

## 5. 后端核心模块

- `src-tauri/src/lib.rs`：应用启动、插件初始化、托盘与命令注册
- `src-tauri/src/database.rs`：SQLite 初始化与迁移
- `src-tauri/src/music/`：扫描、库目录/侧边栏目录、封面与歌词、类型定义
- `src-tauri/src/player.rs`：播放引擎、进度与输出设备控制
- `src-tauri/src/statistics.rs`：播放历史与统计聚合
- `src-tauri/src/toolbox.rs`：批量重命名预览/执行、外部程序打开、刷新目录歌曲
- `src-tauri/src/error.rs`：统一错误类型

## 6. 数据结构与存储

数据库位置：`app_data_dir()/library.db`

### 6.1 表结构概览

```sql
CREATE TABLE songs (
  id INTEGER PRIMARY KEY,
  path TEXT NOT NULL UNIQUE,
  title TEXT,
  artist TEXT,
  album TEXT,
  duration INTEGER,
  cover_path TEXT,
  added_at INTEGER,
  bitrate INTEGER,
  sample_rate INTEGER,
  bit_depth INTEGER,
  format TEXT,
  file_size INTEGER,
  file_modified_at INTEGER
);

CREATE TABLE library_folders (
  path TEXT PRIMARY KEY,
  added_at INTEGER
);

CREATE TABLE sidebar_folders (
  path TEXT PRIMARY KEY,
  added_at INTEGER
);

CREATE TABLE play_history (
  id INTEGER PRIMARY KEY,
  song_path TEXT NOT NULL,
  played_at INTEGER NOT NULL,
  event TEXT DEFAULT 'play',
  played_seconds INTEGER DEFAULT 0,
  song_id INTEGER
);
```

### 6.2 索引

- `idx_songs_added_at`：按入库时间查询
- `idx_play_history_played_at`：按播放时间区间查询
- `idx_play_history_song_id`：按歌曲统计

## 7. 关键流程

### 7.1 音乐库扫描

1. 前端调用 `scan_library` 或 `scan_music_folder` 等 Tauri 命令。
2. `music/scanner.rs` 使用 `walkdir` 遍历目录并读取元数据（lofty）。
3. 解析结果写入 `songs`，并更新 `library_folders` 或 `sidebar_folders`。
4. 前端刷新列表，`song-list` 组件通过虚拟列表渲染大量条目。

### 7.2 播放流程

1. 前端调用 `play_audio(path)`。
2. `player.rs` 使用 Rodio 加载并播放，更新进度与设备状态。
3. 前端同步封面与歌词请求，更新播放器 UI 与动态背景。

### 7.3 统计流程

1. 播放事件写入 `play_history`。
2. `statistics.rs` 聚合行为、格式与质量分布。
3. 前端统计页组件渲染图表与概览卡片。

### 7.4 工具箱：批量重命名

1. 设置页打开工具箱（`SettingsToolbox.vue`）。
2. 调用 `preview_rename` 生成预览与变更清单。
3. 调用 `apply_rename` 执行批量改名，必要时刷新目录歌曲。

## 8. 开发提示

- 新增页面：在 `src/views/` 新建视图文件，并在 `src/router/index.ts` 注册。
- 新增 Tauri 命令：在 `src-tauri/src` 实现 `#[tauri::command]` 并在 `lib.rs` 中注册。
- 修改数据库结构：在 `database.rs` 中添加迁移逻辑，保持老库兼容。
